# AGENTS.md

## 严格限制：不得改动笔记正文

这个仓库用于存放用户自己的笔记（notes）。这些笔记必须由用户本人完成、整理和表达。Codex 必须严格保持这些笔记的纯净性，不得主动参与笔记正文内容的写作、润色、改写、重组、补全或修改。

后续仓库中的笔记主要会是 Markdown 或 MDX 格式。除非用户明确、具体、直接地要求 Codex 修改某个文件中的对应范围，否则 Codex 绝对不能改动任何笔记内容。这个限制包括但不限于：

- 不得替用户写笔记正文。
- 不得替用户润色、优化、扩写、缩写或改写笔记正文。
- 不得调整用户笔记中的措辞、语气、结构、段落顺序或表达方式。
- 不得因为内容看起来混乱、粗糙、重复、不完整、语法不顺或表达不佳而主动修改。
- 不得在没有明确指令的情况下修复 Markdown 或 MDX 笔记正文里的格式、标点、标题层级、列表样式或排版问题。
- 不得主动添加总结、解释、例子、过渡句、标题、注释或任何正文内容。
- 不得把用户的想法改写成 Codex 的表达方式。

只有在用户明确要求时，Codex 才能修改用户指定的对应范围。例如，用户可能会明确要求在某个 MDX 文件中加入组件、调整某一段指定内容、或修改某个明确范围内的结构。即使在这种情况下，Codex 也必须严格限制修改范围，只处理用户明确指定的内容，不得顺手润色、整理或扩展其他笔记正文。

本仓库中的笔记应当保留用户自己的 idea、想法、思路、语言和表达习惯。即使笔记写得很乱、很烂、很不完整，Codex 也必须保持原样，不得擅自修改。保持笔记内容的 pure（纯净）是本仓库最重要的项目级规则。
